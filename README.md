## 🔍 论文深度解读

### 1. 🪙 Bitcoin Whitepaper 解读
> **核心逻辑**：打造一套不用银行、双方直接转账的电子现金系统，解决“双花”问题。

<details>
<summary><b>📌 点击展开：比特币白皮书精华笔记</b></summary>

#### 一、 核心痛点：传统电子支付的麻烦
* **依赖中介**：必须信任第三方，产生手续费，且小额转账不划算。
* **不可逆性差**：交易可能被撤回，商家需防范诈骗。
* **双花问题**：没有第三方时，难以确认钱是否被重复支付。

#### 二、 关键设计：如何实现“无信任转账”？
1.  **数字签名链**：比特币是一串数字签名形成的 ownership 链，通过私钥签名、公钥验证归属。
2.  **PoW (工作量证明)**：节点竞争计算哈希题，只有胜者能打包区块。
3.  **最长链机制**：全网只认“最长的链”，篡改历史需要超过全网 51% 的算力。
4.  **激励机制**：通过“铸币奖励”和“交易费”吸引节点维护安全。

#### 三、 总结
比特币是全网共同维护、用算力担保、不可篡改的公开账本。

### 2. 💎 Ethereum Whitepaper 解读
> **核心逻辑**：将区块链从“计算器”升级为“操作系统”，支持图灵完备的智能合约。

<details>
<summary><b>📌 点击展开：以太坊白皮书精华笔记</b></summary>

#### 一、 比特币的“局限”
* **功能单一**：仅能转账，无法实现复杂逻辑（如自动分红）。
* **状态缺失**：无法记录除了币归属以外的其他信息（如合约到期时间）。

#### 二、 以太坊的核心目标
* **图灵完备语言**：内置编程语言，能实现任何复杂的业务逻辑。
* **状态机模型**：不仅记录余额，还记录合约的中间状态。

#### 三、 关键概念
* **账户体系**：分为外部账户（人控制）和合约账户（代码控制）。
* **智能合约**：无人看管的自动机器，代码即法律。
* **Gas (燃料)**：每步操作消耗 Gas，防止恶意循环拖垮网络。

#### 四、 总结
比特币是去中心化货币，以太坊是去中心化应用平台，让“用代码代替规则”成为可能。

### 3. 🌐 IPFS Whitepaper 解读
> **核心逻辑**：打造分布式、永久、高效的全球文件系统，解决 HTTP 的“单点故障”。

<details>
<summary><b>📌 点击展开：IPFS 白皮书精华笔记</b></summary>

#### 一、 IPFS 想解决什么问题？
* **中心化脆弱**：服务器宕机则文件丢失（404 问题）。
* **带宽浪费**：重复从中心服务器拉取相同数据。

#### 二、 核心技术整合
* **BitTorrent 模式**：点对点互传，热门文件下载更快。
* **Git 模式**：用哈希值（指纹）唯一标识文件，天然支持版本回溯。
* **DHT (分布式哈希表)**：通过文件指纹在全球网络中快速定位资源。
* **BitSwap 协议**：文件交换市场，鼓励分享，防止“白嫖”。

#### 三、 文件存储逻辑
1.  **Merkle DAG 结构**：大文件拆成片段互相链接，改动局部不影响整体。
2.  **内容寻址**：地址是 `/ipfs/[文件指纹]` 而非网址。只要有人存，地址永不失效。

#### 四、 总结
IPFS 致力于将“脆弱、中心化”的互联网，变成“坚固、高效”的分布式网络。
